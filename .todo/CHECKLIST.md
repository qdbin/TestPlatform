# AI 相关模块 MVP 验收清单（待办）

## 需求分析

### 需求符合性检查

- [ ] 知识库管理不在左侧边栏嵌入，管理界面为弹窗
- [ ] 用例生成仅针对项目存在接口；步骤只保存 apiId（写入 case_api.api_id）
- [ ] 不存在接口：先引导用户进入接口生成并复用接口新增组件渲染，用户保存后，再根据上下文需求引导用户是否生成用例，确认后再生成用例
- [ ] 用例生成预览可编辑，复用用例新增组件；不使用纯 JSON `<pre>` 预览
- [ ] 对话为 SSE 真流式输出（token 级增量），前端实时渲染
- [ ] 切换到其他页面不会自动中断对话（默认策略）
- [ ] 不出现“relativeURL.replace is not a function”
- [ ] Embedding 可用且 RAG 走向量检索（非静默回退）
- [ ] AI 模块不影响现有非 AI 功能模块行为

### 风险与回归边界检查

- [ ] AI 相关改动仅落在 aiAssistant、AiController/AiService、ai-service 目录内（以及必要的DB迁移/字段）
- [ ] 不新增全局拦截器/全局路由守卫副作用
- [ ] 不在日志/前端暴露 API-Key

## 设计实现

### 架构与数据流验证

- [ ] 架构图与实现一致：FE -> BE -> FastAPI -> LLM/Embeddings/Chroma
- [ ] 对话数据流：LLM stream -> FastAPI SSE -> SpringBoot SSE -> 前端 reader 实时追加
- [ ] 知识库数据流：索引时向量入库成功才标记 indexed；检索返回引用片段可核对来源
- [ ] 用例生成数据流：selectedApis(apiId[]) -> 生成 CaseRequest -> 灌入用例新增组件 -> /case/save 真保存

### 功能验收用例（手工 + 脚本）

#### 1) 对话流式

- [ ] 发送较长问题，观察回答内容逐步增长（非一次性出现）
- [ ] 流式过程中点击“停止”，可立即终止并可继续发送下一条
- [ ] 切换到其他页面（如接口管理）再切回，生成不中断或能继续展示输出

#### 2) 知识库（弹窗管理）

- [ ] 打开知识库管理弹窗：目录树加载正常
- [ ] 新建目录/新建文档：保存成功后目录树刷新正常
- [ ] 查看/编辑文档：内容可正确读写
- [ ] 重建索引：成功/失败/降级状态明确展示（含错误原因）
- [ ] 删除目录：有子节点时禁止删除；无子节点时可删除

#### 3) RAG（向量检索）

- [ ] 索引后问答可命中文档片段（答案带引用片段/来源标识）
- [ ] Embedding 状态可通过诊断接口/界面确认可用
- [ ] 人为制造 Embedding 不可用时：索引返回 degraded 并明确失败原因，且不标记 indexed

#### 4) 用例生成（可编辑 + 真落库）

- [ ] 选择项目中已存在的接口生成用例：预览为可编辑表单
- [ ] 修改步骤描述/断言/参数后保存：后端真实返回成功
- [ ] 数据库校验：`case_api.api_id` 均为有效接口ID且归属当前项目

#### 5) 不存在接口流程

- [ ] 输入需求涉及不存在接口：系统进入“接口生成”流程并渲染接口新增组件
- [ ] 用户保存接口成功后：自动回填新 apiId 并可继续用例生成

### 自动化测试门禁

- [ ] ai-service：pytest 全量通过（包含新增单测）
- [ ] platform-backend：编译通过；AI 相关单测通过
- [ ] platform-frontend：AI 页核心路径无控制台错误（对话/知识库/用例生成）
