# LiuMa前端技术文档总览

## 项目简介
LiuMa是一款自动化测试平台的前端项目，基于Vue.js 2.x框架开发，提供完整的测试生命周期管理功能。平台采用前后端分离架构，支持API、WEB、APP三种测试类型的统一管理。

## 技术架构概览

### 1. 前端框架层（核心基础）
- **Vue.js 2.x**: 渐进式JavaScript框架（英文：Progressive JavaScript Framework）
- **Vue Router**: 官方路由管理器，实现单页面应用（SPA）的页面跳转
- **Vuex**: 状态管理模式，集中管理应用的所有组件状态

### 2. UI组件层（界面构建）
- **Element UI**: 基于Vue的组件库，提供丰富的UI组件
- **ECharts**: 百度开源的数据可视化图表库
- **vue2-ace-editor**: 代码编辑器组件，支持语法高亮

### 3. 工具库层（功能增强）
- **Axios**: 基于Promise的HTTP客户端，用于与后端API通信
- **jQuery**: JavaScript工具库，简化DOM操作
- **SortableJS**: 拖拽排序库，实现列表拖拽功能

### 4. 构建工具层（工程化）
- **Webpack**: 模块打包工具，将各种资源打包成浏览器可识别的格式
- **Babel**: JavaScript编译器，将ES6+代码转换为向后兼容的JavaScript版本
- **ESLint**: 代码检查工具，保证代码质量和风格一致性

### 5. CSS预处理层（样式管理）
- **Sass/SCSS**: CSS预处理器，提供更强大的样式编写能力
- **PostCSS**: 用JavaScript转换CSS的工具

## 功能模块架构

### 核心业务模块
1. **登录认证模块** (`/src/views/login.vue`)
   - 用户登录验证
   - 权限信息获取
   - 路由权限控制

2. **首页看板模块** (`/src/views/home/`)
   - 数据统计卡片
   - 用例执行趋势图
   - 测试报告统计

3. **公共组件中心** (`/src/views/baseCenter/`)
   - 文件管理：测试文件上传下载
   - 公共参数：全局参数配置
   - 函数管理：自定义测试函数
   - 操作管理：UI自动化操作定义

4. **环境中心** (`/src/views/envCenter/`)
   - 环境管理：测试环境配置
   - 引擎管理：测试引擎监控
   - 设备管理：测试设备控制

5. **用例中心** (`/src/views/caseCenter/`)
   - 接口管理：API接口定义
   - 用例管理：测试用例生命周期
   - 用例编辑：API/WEB/APP用例编排

6. **计划管理中心** (`/src/views/planCenter/`)
   - 测试集合：用例批量组织
   - 测试计划：执行策略配置
   - 定时任务：Cron表达式调度

7. **测试报告模块** (`/src/views/report/`)
   - 执行监控：实时进度展示
   - 报告详情：详细结果分析

8. **系统管理模块** (`/src/views/system/`)
   - 项目管理：多项目支持
   - 角色管理：权限角色定义
   - 用户管理：系统用户维护

## 快速开始

### 环境准备
```bash
# 安装Node.js（版本>=6.0.0）和npm（版本>=3.0.0）
node -v
npm -v

# 克隆项目并安装依赖
cd LiuMa-frontend
npm install
```

### 开发启动
```bash
# 启动开发服务器
npm run dev

# 项目运行在 http://localhost:8080
```

### 生产构建
```bash
# 构建生产版本
npm run build

# 构建文件在dist目录
```

## 技术选型原因

### 为什么选择Vue.js？
- **学习曲线平缓**：相比React和Angular更容易上手
- **中文文档完善**：对中文开发者更友好
- **组件化开发**：提高代码复用性和维护性
- **生态系统丰富**：拥有完整的工具链支持

### 为什么选择Element UI？
- **企业级组件库**：专为后台管理系统设计
- **组件丰富完整**：覆盖大部分业务场景
- **文档详细清晰**：提供完整的使用示例
- **社区活跃度高**：问题响应及时

### 为什么选择Webpack？
- **模块化打包**：支持各种资源类型
- **插件生态丰富**：满足各种构建需求
- **配置灵活**：可根据项目特点定制
- **社区支持好**：文档和教程丰富

## 后续学习建议
