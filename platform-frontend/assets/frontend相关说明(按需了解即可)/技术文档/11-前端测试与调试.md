# 前端测试与调试

## 什么是前端测试与调试？

前端测试与调试是确保代码质量和功能正确性的重要环节。

**英文原意**：Frontend Testing & Debugging
**中文理解**：找出代码中的bug并确保功能按预期工作

**生活类比**：就像汽车出厂前的质检和路试，确保每个零件都正常工作，整车性能达标。

## 测试的重要性

### 1. 质量保证
- **减少bug**：在开发阶段发现并修复问题
- **提升稳定性**：确保代码在各种环境下正常工作
- **增强信心**：修改代码时不用担心破坏现有功能

### 2. 开发效率
- **快速定位问题**：测试失败能快速定位错误位置
- **减少回归测试**：自动化测试替代手工测试
- **持续集成**：代码提交自动运行测试

### 3. 团队协作
- **文档作用**：测试用例描述功能预期行为
- **代码规范**：测试驱动开发促进良好设计
- **新人上手**：通过测试了解系统功能

```javascript
// 被测试的函数
// src/utils/format.js
/**
 * 格式化日期
 * @param {Date|string} date - 日期对象或字符串
 * @param {string} format - 格式字符串
 * @returns {string} 格式化后的日期
 */
export function formatDate(date, format = 'YYYY-MM-DD HH:mm:ss') {
  if (!date) return ''
  
  const d = new Date(date)
  if (isNaN(d.getTime())) return ''
  
  const year = d.getFullYear()
  const month = String(d.getMonth() + 1).padStart(2, '0')
  const day = String(d.getDate()).padStart(2, '0')
  const hours = String(d.getHours()).padStart(2, '0')
  const minutes = String(d.getMinutes()).padStart(2, '0')
  const seconds = String(d.getSeconds()).padStart(2, '0')
  
  return format
    .replace('YYYY', year)
    .replace('MM', month)
    .replace('DD', day)
    .replace('HH', hours)
    .replace('mm', minutes)
    .replace('ss', seconds)
}

/**
 * 格式化文件大小
 * @param {number} bytes - 字节数
 * @returns {string} 格式化后的文件大小
 */
export function formatFileSize(bytes) {
  if (bytes === 0) return '0 B'
  
  const k = 1024
  const sizes = ['B', 'KB', 'MB', 'GB', 'TB']
  const i = Math.floor(Math.log(bytes) / Math.log(k))
  
  return parseFloat((bytes / Math.pow(k, i)).toFixed(1)) + ' ' + sizes[i]
}
```

```javascript
class ErrorHandler {
  constructor() {
    this.errors = []
    this.maxErrors = 100
    this.init()
  }
  
  init() {
    // 全局错误捕获
    window.addEventListener('error', (event) => {
      this.handleError({
        type: 'javascript',
        message: event.message,
        filename: event.filename,
        lineno: event.lineno,
        colno: event.colno,
        error: event.error,
        timestamp: Date.now()
      })
    })
    
    // Promise错误捕获
    window.addEventListener('unhandledrejection', (event) => {
      this.handleError({
        type: 'promise',
        message: event.reason?.message || 'Unhandled promise rejection',
        stack: event.reason?.stack,
        timestamp: Date.now()
      })
    })
    
    // Vue错误处理
    if (window.Vue) {
      window.Vue.config.errorHandler = (error, vm, info) => {
        this.handleError({
          type: 'vue',
          message: error.message,
          stack: error.stack,
          component: vm?.$options?.name,
          info: info,
          timestamp: Date.now()
        })
      }
    }
  }
  
  handleError(errorInfo) {
    // 添加到错误列表
    this.errors.push(errorInfo)
    
    // 限制错误数量
    if (this.errors.length > this.maxErrors) {
      this.errors.shift()
    }
    
    // 开发环境下打印详细信息
    if (process.env.NODE_ENV === 'development') {
      console.group('🐛 Error Captured')
      console.error('Type:', errorInfo.type)
      console.error('Message:', errorInfo.message)
      console.error('Stack:', errorInfo.stack)
      console.error('Timestamp:', new Date(errorInfo.timestamp))
      
      if (errorInfo.component) {
        console.error('Component:', errorInfo.component)
      }
      
      if (errorInfo.info) {
        console.error('Info:', errorInfo.info)
      }
      
      console.groupEnd()
    }
    
    // 发送错误到监控平台
    this.reportError(errorInfo)
  }
  
  reportError(errorInfo) {
    // 发送到后端或第三方监控服务
    if (window.g_reportError) {
      window.g_reportError({
        ...errorInfo,
        url: window.location.href,
        userAgent: navigator.userAgent,
        timestamp: Date.now()
      })
    }
  }
  
  // 获取最近的错误
  getRecentErrors(count = 10) {
    return this.errors.slice(-count)
  }
  
  // 清除错误历史
  clearErrors() {
    this.errors = []
  }
  
  // 导出错误报告
  exportErrorReport() {
    const report = {
      timestamp: Date.now(),
      url: window.location.href,
      userAgent: navigator.userAgent,
      errors: this.errors,
      summary: {
        total: this.errors.length,
        byType: this.errors.reduce((acc, error) => {
          acc[error.type] = (acc[error.type] || 0) + 1
          return acc
        }, {})
      }
    }
    
    // 下载报告
    const blob = new Blob([JSON.stringify(report, null, 2)], {
      type: 'application/json'
    })
    
    const url = URL.createObjectURL(blob)
    const a = document.createElement('a')
    a.href = url
    a.download = `error-report-${Date.now()}.json`
    a.click()
    
    URL.revokeObjectURL(url)
  }
}
```

```javascript
    const url = URL.createObjectURL(blob)
    const a = document.createElement('a')
    a.href = url
    a.download = `debug-logs-${Date.now()}.json`
    a.click()
    
    URL.revokeObjectURL(url)
  }
  
  clearLogs() {
    this.logs = []
    this.updateDebugPanel()
  }
  
  hide() {
    const panel = document.getElementById('production-debug-panel')
    if (panel) {
      panel.style.display = 'none'
    }
  }
}

// 创建全局调试器
window.productionDebugger = new ProductionDebugger()

// 快捷键启用调试模式
document.addEventListener('keydown', (e) => {
  // Ctrl + Shift + D
  if (e.ctrlKey && e.shiftKey && e.key === 'D') {
    e.preventDefault()
    window.productionDebugger.enable()
  }
})
```

## 常见问题解答

### Q1：测试和调试有什么区别？

**A**：
- **测试**：主动验证代码是否按预期工作，包括单元测试、集成测试等
- **调试**：被动查找和修复代码中的问题，通常在测试失败或出现bug时进行

**类比**：测试像是定期体检，调试像是生病后的诊断治疗。

### Q2：如何写好的测试用例？

**A**：
1. **清晰明确**：测试目的和预期结果要明确
2. **独立可重复**：每个测试用例应该独立运行
3. **覆盖边界**：测试正常情况和边界条件
4. **命名规范**：测试名称要描述清楚测试内容
5. **保持简单**：测试代码要简单易懂

### Q3：什么时候需要写测试？

**A**：
- **核心功能**：业务逻辑复杂的核心功能
- **公共组件**：被多个地方使用的组件
- **工具函数**：纯函数和工具类
- **API接口**：后端接口对接部分
- **用户流程**：关键的用户操作流程

### Q4：如何调试异步代码？

**A**：
```javascript
// 1. 使用async/await
async function debugAsync() {
  try {
    console.log('开始异步操作')
    const result = await fetchData()
    console.log('异步结果:', result)
  } catch (error) {
    console.error('异步错误:', error)
  }
}

// 2. 使用Promise链式调用
function debugPromise() {
  console.log('开始Promise操作')
  
  fetchData()
    .then(result => {
      console.log('Promise结果:', result)
      return processData(result)
    })
    .then(processed => {
      console.log('处理结果:', processed)
    })
    .catch(error => {
      console.error('Promise错误:', error)
    })
}

// 3. 使用调试器
async function debugWithDebugger() {
  debugger // 设置断点
  
  const result = await complexAsyncOperation()
  console.log(result)
}
```

### Q5：如何处理难以重现的bug？

**A**：
1. **收集信息**：记录用户操作步骤、环境信息
2. **添加日志**：在关键位置添加详细的日志输出
3. **错误监控**：使用错误监控工具收集错误信息
4. **用户反馈**：建立用户反馈渠道收集问题报告
5. **环境模拟**：尽量复现用户的使用环境
