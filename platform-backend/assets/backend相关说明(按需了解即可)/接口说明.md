# 接口说明文档

## 接口通用说明

### 认证机制
- 大部分接口需要在请求头中添加 `token` 进行身份认证
- token通过登录接口获取，在响应头中返回
- 未特殊说明的接口均需要token认证

### 通用响应格式
- 成功响应：直接返回业务数据
- 错误响应：返回错误信息，HTTP状态码非200

## 1. 用户认证模块

### 用户登录：`POST /autotest/login`

**描述**
- 用户登录接口，验证用户账号密码，返回登录用户信息

**请求头**：无

**请求数据**:
```json
{
  "account": "admin",        // 用户账号
  "password": "123456"       // 用户密码
}
```

**响应头**:
```
token: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...  // JWT令牌
```

**响应数据**:
```json
{
  "id": "user123",           // 用户ID
  "username": "管理员",       // 用户名称
  "account": "admin",        // 用户账号
  "status": "active",        // 用户状态
  "createTime": 1642694400,  // 创建时间
  "updateTime": 1642694400,  // 更新时间
  "mobile": 13800138000,     // 手机号
  "lastProject": "proj123",  // 最后访问项目ID
  "email": "admin@test.com", // 邮箱
  "permissions": ["perm1", "perm2"] // 权限列表
}
```

### 用户注册：`POST /autotest/register`

**描述**
- 用户注册接口，创建新用户账号

**请求头**：无

**请求数据**:
```json
{
  "username": "新用户",      // 用户名称
  "account": "newuser",      // 用户账号
  "password": "123456",      // 用户密码
  "mobile": 13800138000,     // 手机号
  "email": "user@test.com"   // 邮箱
}
```

**响应数据**:
```json
"注册成功"  // 注册结果消息
```
